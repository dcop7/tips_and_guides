# 🧠 Personal Productivity — Reference Card

---

## ⭐ The Rule

| 📝 Notebook | ✅ Todoist | 🧱 Notion |
|-------------|-----------|----------|
| What I do **now** | Everything I need **to do** | What I need **to remember** |

> If something doesn't fit cleanly into one of these, the system is being misused — not that it needs more complexity.

---

## 🗂️ Where does this go?

| What | Where |
|------|-------|
| ✅ Task or action | Todoist |
| 📝 Meeting note (live) | Notebook |
| 💡 Quick idea, no context | Notion Inbox |
| 📌 Belongs to a Topic | Topic scratchpad or sub-page |
| 🔧 Technical reference | Knowledge Base → Tech Reference |
| ⚙️ Process or workflow | Knowledge Base → Processes |
| 🗑️ Not worth keeping | Discard |

---

## 📎 Note inside a Topic?

| Type | Where |
|------|-------|
| 1–3 lines, no structure | 💬 Scratchpad — `2026-05-07 — note` |
| Long, structured, has a title | 📄 Sub-page inside the Topic row |

---

## 🤝 After a meeting

1. 🔲 Extract tasks → **Todoist**
2. 💬 Extract decisions and context → **Topic scratchpad**
3. 📋 Full meeting notes → **Meetings DB row**, linked to Topic

---

## 📅 Daily routine

### 🌅 Morning
- Review Todoist → pick Big 3 → write in Notebook
- Glance at Topics DB — anything ending soon?

### ☀️ During the day
- Work from the Notebook Big 3
- Capture tasks in Todoist as they appear
- Raw meeting notes in Notebook

### 🌙 End of day
- Process Todoist inbox
- Process Notebook notes → Notion
- Clear Notion Inbox

---

## 🔁 Weekly review

- 🟢 Topics Done > 2 weeks → confirm filtered from main view
- ⏸️ On Hold Topics → still relevant?
- 🗑️ Todoist P3/P4 sitting too long → delete or reschedule
- 📥 Notion Inbox → anything leftover?
- 🗄️ Archive → move loose old pages to the right year folder

---

## ✅ Todoist priorities

| Priority | Meaning |
|----------|---------|
| 🔴 P1 | Urgent — do today |
| 🟠 P2 | Important — do this week |
| 🟡 P3 | Nice to have |
| ⚪ P4 | Someday / maybe |

💡 **Deadlines in the title:** `Fix auth bug [by 20 May]`
💡 **Context in the title:** `[API redesign] Update endpoint docs`
